Nginx服务器ssl证书配置

获取ssl证书

本网站使用阿里云提供服务。

在阿里的控制台搜索ssl证书界面，即可每年领取20个证书的份额，每个证书现在有效期是3个月。

填写申请域名材料即可申报一个证书。得到证书之后按照指示下载.key公钥文件和.pem私钥文件。

配置ssl证书

第一步 放置证书

首先在一个权限安全的地方存放好密钥文件，位置随意。

第二步 更改nginx的配置文件

找到配置文件

cd /etc/nginx/sites-available/default

写入

server {
    #HTTPS的默认访问端口443。
    #如果未在此处配置HTTPS的默认访问端口，可能会造成Nginx无法启动。
    listen 443 ssl;

    #填写证书绑定的域名
    server_name newland.khvemx.work;

    #填写证书文件绝对路径
    ssl_certificate path/khvemx.work.pem;
    #填写证书私钥文件绝对路径
    ssl_certificate_key path/khvemx.work.key;

    ssl_session_cache shared:SSL:1m;
    ssl_session_timeout 5m;

    #自定义设置使用的TLS协议的类型以及加密套件（以下为配置示例，请您自行评估是否需要配置）
    #TLS协议版本越高，HTTPS通信的安全性越高，但是相较于低版本TLS协议，高版本TLS协议对浏览器的兼容性较差。
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
    ssl_protocols TLSv1.1 TLSv1.2 TLSv1.3;

    #表示优先使用服务端加密套件。默认开启
    ssl_prefer_server_ciphers on;
 
    location / {
        root path/html;
        index index.html;

        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.1-fpm.sock;
    }
}

第三步 重定位HTTP服务

如果想让来自80端口的http请求重定位到https上，写入

server {
    listen 80;

    server_name newland.khvemx.work;

    rewrite ^(.*)$ https://newland.khvemx.work;

    location / {
        index index.html index.htm;
    }
}

第四步 检查和重启

然后回到bash中检查一下有无错误

nginx -t

如果没有错误就重启nginx服务，此时ssl就成功部署到nginx上了

systemctl restart nginx
